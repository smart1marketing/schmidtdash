ECWID DASHBOARD COMPLETE PACKAGE MANIFEST
==========================================

This package contains everything needed to deploy a production-ready 
Ecwid ecommerce analytics dashboard to Render and embed it in HighLevel.

## TOTAL FILES: 9

## FILE LISTING WITH DESCRIPTIONS

### 1. server.js (ESSENTIAL)
   Purpose: Main Node.js/Express backend server
   Size: ~350 lines of code
   What it does:
   - Integrates with Ecwid API
   - Fetches orders and products
   - Calculates metrics (order counts, changes)
   - Stores data in JSON file
   - Serves data via REST API endpoints
   - Auto-schedules daily refresh at 2 AM UTC
   
   Key Functions:
   - fetchOrdersForDateRange() - Query orders from Ecwid
   - fetchTopProducts() - Get top 10 selling products
   - fetchAllData() - Main data fetch orchestrator
   - /api/dashboard - API endpoint for widget
   - /api/fetch-now - Manual refresh endpoint
   - /health - Service health check

### 2. package.json (ESSENTIAL)
   Purpose: Node.js project configuration and dependencies
   Size: ~30 lines
   Contains:
   - Project metadata
   - npm dependencies (express, axios, node-cron)
   - npm scripts (npm start)
   - Node.js version requirement (18.x)
   
   Dependencies:
   - express@^4.18.2 - Web framework
   - axios@^1.6.0 - HTTP client for API calls
   - node-cron@^3.0.2 - Cron job scheduling

### 3. .gitignore (ESSENTIAL)
   Purpose: Prevents sensitive files from being committed to GitHub
   Size: ~15 lines

   - node_modules/ - npm dependencies folder

   - Logs files

   - OS-specific files
   
   ⚠️ CRITICAL: Ensures API tokens never leak to GitHub

## KEY CONFIGURATIONS

### Ecwid API Integration
- Endpoint: https://api.ecwid.com/v3/{STORE_ID}/...
- Authentication: Bearer token
- Scopes needed: read_orders, read_products
- No cost for API calls

### Render Deployment
- Service: Web Service
- Runtime: Node.js 18.x
- Build: npm install
- Start: npm start
- Schedule: Daily at 2 AM UTC
- Cost: Free tier available

### Data Storage
- Format: JSON
- Location: data/dashboard-data.json
- Refresh: Daily at 2 AM UTC
- Retention: Latest data only
- Backup: GitHub version control
